# bazos-parcer

![poster](image_for_readme.png)

Telegram-бот, который мониторит [reality.bazos.sk](https://reality.bazos.sk) и шлёт новые объявления об аренде квартир по твоим фильтрам.

## Что умеет

- Парсит выдачу `reality.bazos.sk/prenajmu/byt/` с фильтрами (PSC, радиус, цена, текст).
- Хранит увиденные объявления в SQLite — каждое объявление шлётся один раз.
- Фильтры настраиваются через меню в Telegram, без редеплоя.
- На первом подключении не флудит — текущая выдача помечается как просмотренная, шлются только новые.

## Управление

Всё через **inline-меню**. Команд минимум:

| Команда | Что делает |
|---|---|
| `/start` | Подписаться + открыть меню |
| `/menu` | Открыть меню |
| `/check` | Быстрая проверка |
| `/filters` | Текущие фильтры |
| `/stop` | Отписаться |
| `/help` | Помощь |

### Меню

- **Проверить сейчас** — форс-проверка вне расписания.
- **Фильтры** — список фильтров. Нажатие на любой открывает редактор:
  - кнопки-пресеты (радиус, цена, PSC),
  - **Ввести вручную** — следующим сообщением, бот валидирует и сохранит.
- **Статистика** — тиков, отправлено, ошибок, дата последней проверки, счётчики по статусам.
- **Пауза / Возобновить** — приостановить уведомления.
- **Сбросить кеш** — забыть все просмотренные (с подтверждением).
- **Отписаться** — удалить все данные (с подтверждением).
- **Отписанные** — список объявлений, которым ты написал, с пагинацией.
- **Не нравится** — список отбракованных.

### Под каждым объявлением

Объявление приходит как альбом из всех фото (до 10 шт.), затем отдельным сообщением — кнопки действий:

- **Открыть на bazos** — прямая ссылка на страницу объявления. Там можно раскрыть телефон (требует SMS-верификации на стороне bazos, авто-получение не работает).
- **Отписал** — отметить, что уже написал или позвонил. Кнопка помечается галкой.
- **Не нравится** — отправить в список отбракованных.
- **Сбросить** — снять статус.

Имя автора парсится с детальной страницы и отображается под описанием.

### Поля фильтров

| Ключ | Описание | Допустимо |
|---|---|---|
| `hlokalita` | PSC (почтовый индекс) | 5 цифр, e.g. `04001` |
| `humkreis` | Радиус, км | 0/1/2/5/10/20/30/50/75/100 |
| `cenaod` | Цена от, EUR | число или пусто |
| `cenado` | Цена до, EUR | число или пусто |
| `hledat` | Текстовый поиск | до 100 символов |
| `order` | Сортировка | пусто (по дате), `1` (дешевле), `2` (дороже) |

## Локальный запуск

```bash
pip install -r requirements.txt

# Получи токен у @BotFather в Telegram
export TELEGRAM_BOT_TOKEN=123456:ABC...
export CHECK_INTERVAL_SEC=600
export DB_PATH=./bazos.sqlite3

python bot.py
```

Открой бота в Telegram → `/start`.

## Деплой на fly.io

> Хватит бесплатного триала (7 дней). После — ~$1.94/мес за shared-cpu-1x@256MB + $0.15/GB volume.

```bash
# 1. Установи flyctl
# Windows: iwr https://fly.io/install.ps1 -useb | iex
# Mac/Linux: curl -L https://fly.io/install.sh | sh

fly auth login

# 2. В корне проекта
fly launch --no-deploy --copy-config
# App name: придумай уникальный, e.g. bazos-byt-notifier-abc
# Region: fra
# Postgres/Redis: NO

# 3. Volume для SQLite (персистентное хранилище)
fly volumes create bazos_data --region fra --size 1

# 4. Токен бота как секрет
fly secrets set TELEGRAM_BOT_TOKEN=123456:ABC...

# 5. Деплой
fly deploy
```

### Проверка после деплоя

```bash
fly status     # 1 машина в state=started
fly logs       # ищи: "starting bot, interval=600s"
```

В Telegram: `/start` → бот должен ответить. Через ~10 минут придут первые новые объявления.

## Архитектура

```
bot.py        — Telegram handlers, callback router, JobQueue (polling loop)
menus.py      — inline-keyboard layouts, callback_data conventions
scraper.py    — HTTP fetch + BeautifulSoup парсинг list и detail страниц
storage.py    — SQLite: подписчики, фильтры, seen-ads, статусы, статистика
Dockerfile    — python:3.12-slim
fly.toml      — 1 shared-cpu машина + persistent volume
```

Callback data формат: `<scope>:<action>[:<args>]`, влезает в 64-байтный лимит Telegram:
- `m:*` — главное меню (`m:check`, `m:filters`, `m:pause`, `m:list:contacted:0`, ...)
- `f:edit:<key>` / `f:set:<key>:<val>` / `f:custom:<key>` / `f:clear:<key>`
- `ad:c:<id>` / `ad:d:<id>` / `ad:n:<id>` — contacted / disliked / reset

## Подводные камни

- **Капча bazos** при частом опросе. Дефолт 10 минут — безопасно. Не ставь ниже 5 минут.
- **HTML может поменяться.** Селектор `.inzeraty` стабилен, но парсер лучше мониторить по логам.
- **Телефон** на bazos закрыт SMS-верификацией — только через кнопку "Открыть на bazos".
- **Fly volume** не удаляй при редеплое — там SQLite с историей и фильтрами.
