# bazos-parcer

![poster](image_for_readme.png)

Telegram-бот, который мониторит [reality.bazos.sk](https://reality.bazos.sk) и шлёт новые объявления об аренде квартир по твоим фильтрам.

## Что умеет

- Парсит выдачу `reality.bazos.sk/prenajmu/byt/` с фильтрами (PSČ, радиус, цена, текст).
- Хранит увиденные объявления в SQLite — каждое объявление шлётся один раз.
- Фильтры настраиваются командами в Telegram, не нужно редеплоить.
- На первом подключении не флудит — текущая выдача помечается как просмотренная, шлются только новые.

## Управление

Всё через **inline-меню**. Команд минимум:

| Команда | Что делает |
|---|---|
| `/start` | Подписаться + открыть меню |
| `/menu` | Открыть меню |
| `/check` | Быстрая проверка |
| `/filters` | Текущие фильтры |
| `/stop` | Отписаться |
| `/help` | Помощь |

### Меню

- **🔍 Проверить сейчас** — форс-проверка вне расписания.
- **⚙️ Фильтры** — список фильтров. Тап по любому → редактор:
  - кнопки-пресеты (радиус 1/2/5/10/20/30/50 км, цена 400/500/600/700, PSČ Košice/BA…),
  - **✏️ Ввести вручную** — отправь следующее сообщение, бот валидирует и сохранит.
- **📊 Статистика** — тиков, отправлено, ошибок, дата последней проверки.
- **⏸ Пауза / ▶️ Возобновить** — приостановить уведомления.
- **🧹 Сбросить кэш** — забыть все «просмотренные» (с подтверждением).
- **🛑 Отписаться** — удалить все данные (с подтверждением).

### Под объявлением

Каждое объявление приходит как **альбом из всех фото** (до 10 шт., как позволяет Telegram), потом отдельным сообщением — кнопки:
- **🔗 Открыть на bazos** — deep-link, там можно тапнуть «zobraz číslo» чтобы получить телефон (это требует SMS-верификации, авто получить нельзя).
- **✉️/✅ Отписал** — отметить «я уже написал/позвонил». Кнопка превращается в ✅.
- **👎 Не нравится** — спрятать в отдельный список.
- **↩️ Сбросить** — снять статус.

В главном меню добавились разделы:
- **✅ Отписанные** — список объявлений, кому ты отписал, с пагинацией.
- **👎 Не нравится** — список того, что отбраковал.

В каждом объявлении парсится **имя автора** (👤) с детальной страницы.

### Поля фильтров

| Ключ | Описание | Допустимо |
|---|---|---|
| `hlokalita` | PSČ | 5 цифр, e.g. `04001` |
| `humkreis` | Радиус, км | 0/1/2/5/10/20/30/50/75/100 |
| `cenaod` | Цена от, € | число или пусто |
| `cenado` | Цена до, € | число или пусто |
| `hledat` | Текстовый поиск | до 100 символов |
| `order` | Сортировка | пусто (по дате), `1` (дешевле), `2` (дороже) |

## Локальный запуск

```bash
pip install -r requirements.txt

# 1. получи токен у @BotFather
# 2. положи его в .env (см. .env.example) или экспортируй
export TELEGRAM_BOT_TOKEN=123456:ABC...
export CHECK_INTERVAL_SEC=600
export DB_PATH=./bazos.sqlite3

python bot.py
```

В Telegram открой свой бот → `/start`.

## Деплой на fly.io

> Хватит бесплатного triala. После него — $1.94/мес за shared-cpu-1x@256MB + $0.15/GB SSD volume.

```bash
# 1. установи flyctl: https://fly.io/docs/hands-on/install-flyctl/
fly auth login

# 2. в корне проекта
fly launch --no-deploy --copy-config
# Отвечай:
#   - app name: bazos-byt-notifier-<твой-суффикс> (должен быть глобально уникален)
#   - region: fra (или другой ближайший)
#   - postgres / redis: NO

# 3. создай volume для SQLite (1 GB хватит с запасом)
fly volumes create bazos_data --region fra --size 1

# 4. секрет — токен бота
fly secrets set TELEGRAM_BOT_TOKEN=123456:ABC...

# 5. деплой
fly deploy

# Логи
fly logs
```

### Проверки после деплоя

```bash
fly status            # должен быть 1 машина в state=started
fly logs              # ищи "starting bot, interval=600s"
fly ssh console       # зайти внутрь машины
```

В Telegram отправь боту `/start` — должен ответить и через ~10 минут начать слать новые объявления.

## Архитектура

```
bot.py        — telegram handlers + callback router + JobQueue
menus.py      — inline-keyboard layouts + callback_data conventions
scraper.py    — fetch + BeautifulSoup парсинг bazos
storage.py    — SQLite: подписчики, фильтры, seen-ads, статистика
Dockerfile    — python:3.12-slim
fly.toml      — 1 shared-cpu машина + persistent volume
```

### Callback data

Все кнопки используют короткий формат `<scope>:<action>[:<args>]` чтобы влезать в 64-байтный лимит Telegram:
- `m:*` — main menu actions (`m:check`, `m:filters`, `m:pause`, `m:reset:yes`, …)
- `f:edit:<key>` / `f:set:<key>:<val>` / `f:custom:<key>` / `f:clear:<key>`
- `ad:hide:<ad_id>` — скрыть объявление

Всё в одном процессе. Один бот = один пользователь обычно, но multi-user поддерживается — каждый chat_id хранит свои фильтры независимо.

## Подводные камни

- **bazos может отдавать капчу при частом опросе.** Дефолт 10 минут — безопасно. Не уменьшай ниже 5 минут.
- **HTML может поменяться.** Селектор `.inzeraty` стабилен много лет, но если поломается — смотри `bazos_sample.html` для актуальной структуры.
- **Telegram throttling.** Лимит 30 msg/sec на бота — не достижим в нашем кейсе (до 10 объявлений за тик).
- **Fly free trial:** машину нельзя останавливать (`auto_stop=false`), иначе пропустишь объявления. 256MB RAM хватает с запасом.
